# System Design Principles for ML-Powered Applications

When designing ML systems like RAG, some key principles are:

- Design before coding: define components, data flow, and interfaces.
- Separate concerns: retrieval, reranking, orchestration, and serving should be modular.
- Measure everything: model metrics, system metrics, and business metrics.
- Start simple: build a minimal, reliable baseline before adding complexity.
- Optimize for observability: logs, traces, and metrics should make debugging easy.

These principles help ensure that ML systems are maintainable, debuggable, and production-ready.
